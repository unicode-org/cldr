LDML Canonical Form

MED, 2004-12-21

To allow for simple comparison of LDML files, especially for vetting mechanical changes, we should have a canonical form for those files. Here is a proposal. [TBD, add examples.]

  1. All start elements are on their own line, indented by depth tabs. Any leaf node with empty content is in the form <foo/>. All end elements (except for leaf nodes) are on their own line, indented by depth tabs. There are no blank lines except within comments.
  2. All attribute values use ", not '. There are no CDATA sections, no NCRs, and minimal other forms (eg no &apos; since it is not necessary.
  3. Elements are ordered by the Element Order Table
  4. Attributes are ordered by the Attribute Order Table
  5. Values are a bit more complicated, and may depend on the attribute. Compare two values by using the following steps.
    1. If two values are in the table, compare according to the order in the table. Otherwise if one is, it is first.
    2. If two values are numeric [0-9], compare numerically ("2" < "12"). Otherwise if one is numeric, it is first.
    3. Time zone IDs are sorted by GMT order, from smallest to largest. Use the GMT offset as of Jan 1, 2004 first, then the GMT offset as of July 1, 2004 next, otherwise if both are equal, use alphabetical order.
  6. Comments are logically attached to a node. Their contents is indented one further level. There are three kinds:
    1. Inline always appear after the leaf node, starting on the same line.
    2. Block comments always precede the node, and are indented on the same level.
    3. Multiline comments, whether inline or block, have '-->' on a separate line,
