Latest Release Version

The latest version of CLDR is Version 1.2. This new release contains data for 232 locales, covering 72 languages and 108 territories. There are also 63 draft locales in the process of being developed, covering an additional 27 languages and 28 territories.

In this release, the major additions to CLDR are:

  • Data: Many names for languages, territories, and scripts have been added, as well as for time zones, calendars, and other named items such as collation; the data has been compacted by removal of inherited or aliased data; plus other fixes and additions.
  • Structure: The LDML specification has been enhanced substantially. It now has self-contained descriptions of date / number / choice format patterns, inheritance and validity, time zone fallbacks, and provides lists of all valid attribute values. The XML format adds structure to assist in vetting, to allow for multiple sets of exemplar characters (indicating characters in use in particular locales), to represent relative dates and times, to provide better support for time zone names, and to strengthen the alias mechanism.
  • Implementation: The new collation tests allow implementations to verify correct use of locale data. For comparison of data, by-type charts and vetting charts were added. The CLDR source now does not require ICU for generation.

The release is in the form of a zip file. It contains three directories, for common data, for collation data, and for tests. The format of the tests is explained in a readMe.html in the folder. To see the list of all bugs fixes and additions for this release, go to Release Changes. This is a view into the feature/bug database that presents only those closed in this release.

This is a stable release and may be used as reference material or cited as a normative reference by other specifications. Although the data will not be changed, bug reports for changes in future versions filed at Bug Reports. The freeze date for the next version of CLDR is January 15, 2005. All new data or defect reports for CLDR 1.3 must be submitted by then.

Latest Draft Version

The latest draft version is in a source-code repository.

The CVS source control repository is currently being hosted by the ICU project as a service to the Unicode Consortium until it can be moved to the Unicode site. The Unicode Consortium thanks the ICU project for this support.

If you want to browse the data and make occasional downloads, you may want to use WebCVS. It provides a convenient, web-based interface for browsing and downloading the latest version of the files highlighting the differences and sources of the locale data. Otherwise you can get a CVS client, as described below.

Note: The current draft version also includes draft test data.

CVS

The release data contains only the common data. Access to the latest working snapshot of CLDR, and access to data collected for other platforms is available through CVS, as described below.

WinCVS

If you will be doing serious work on CLDR, you should probably install a CVS client on your own machine so that you can do batch operations without going through the WebCVS interface. On Windows, we suggest the WinCVS client. The following is the example instruction on how to download CLDR via WinCVS:

  1. Install the WinCVS client, which you can download from the WinCVS home page.
  2. In the WinCVS preferences, specify your CVSRoot to be ":pserver:anoncvs@oss.software.ibm.com:/usr/cvs/icu"  with the password "anoncvs". To enter the CVSRoot value, select "Preferences" from the "Cvs Admin" pull-down menu.  Authentication should be set to "'passwd' file on the cvs server". 
  3. To "extract" the most recent version of CLDR, select "Checkout module" from the "Cvs Admin" menu.  Specify "locale" for the module name.

CVS command line

You can also check out the repository anonymously on UNIX using the following commands, after first setting your CVSROOT to point to the CLDR repository:

  1. export CVSROOT=:pserver:anoncvs@oss.software.ibm.com:/usr/cvs/icu 
  2. cvs login CVS password: anoncvs 
  3. cvs checkout -P locale

This will extract the absolute latest version of all files from the repository. If you want to extract a particular release, say 1.2.3, pass a -r TAG option to cvs checkout. For example:

cvs checkout -P -r release-1-2-3 locale

You should then have a working repository.

Repository Organization

In the descriptions below, <locale_data > is the full path name of the locale data root directory (the top level directory from the distribution archives) in your file system. The directories are named to reflect the source platform from which they are collected; for e.g. locale data collected from ICU is in the directory named “icu”.

The following directories contain source code and data files:

<path>/locale The repository containing locale data from different platforms.
   /<platform_name> Contains locale data collected from the specific platform denoted by the <platform_name>. "Common" is the common data.
/main The locale data in XML format, e.g.: locale/icu/main/en.xml, locale/icu/main/fr_FR.xml.
/collation The collation data in XML format, e.g.: locale/icu/collaiton/en.xml, locale/icu/collation/fr_FR.xml.
    /test for the common data, the conformance tests
/diff Contains comparison charts. The charts show data for different locales across platforms in tabular format/
/main Contains the comparison charts for main data
/collation Contains the comparison charts for collation data

The XML source data is organized in the following directories. The xml files can be divided into main/ and collation/.

Note: the data for all but common is provided for comparison only, and is not to be viewed as authoritative or referenceable.
