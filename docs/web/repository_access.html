Repository Access

The latest version of CLDR is Version 1.1. This release contains roughly 50% more data than CLDR 1.0: adding many translated terms for languages, scripts, countries, currencies, and timezones. It also improves collation for a number of languages in Eastern Europe, and adds stand-alone month data for Slavic languages and Greek, and narrow month/day names for quite a number of languages. Among other things, the LDML 1.1 adds new formats for narrow and stand-alone month and day names  (used in online calendar applications), and POSIX compatibility fields.

The contents of this version are packaged in the following locations:

  • http://www.unicode.org/Public/cldr/cldr1.1.zip
  • ftp://www.unicode.org/Public/cldr/cldr1.1.zip

This is a stable release and may be used as reference material or cited as a normative reference by other specifications. Although the data will not be changed, bug reports for changes in future versions filed at Bug Reports. The next version, CLDR 1.2, is planned for mid-October, but any reports must be filed well ahead of time for consideration.

The release data contains only the common data. Access to the latest working snapshot of CLDR, and access to data collected for other platforms is available through CVS, as described below. (This data is currently being hosted on the ICU site.)

WebCVS

If you want to browse the code and only make occasional downloads, you may want to use WebCVS. It provides a convenient, web-based interface for browsing and downloading the latest version of the files highlighting the differences and sources of the locale data.

WinCVS

If you will be doing serious work on CLDR, you should probably install a CVS client on your own machine so that you can do batch operations without going through the WebCVS interface. On Windows, we suggest the WinCVS client. The following is the example instruction on how to download CLDR via WinCVS:

  1. Install the WinCVS client, which you can download from the WinCVS home page.
  2. In the WinCVS preferences, specify your CVSRoot to be ":pserver:anoncvs@oss.software.ibm.com:/usr/cvs/icu"  with the password "anoncvs". To enter the CVSRoot value, select "Preferences" from the "Cvs Admin" pull-down menu.  Authentication should be set to "'passwd' file on the cvs server". 
  3. To "extract" the most recent version of CLDR, select "Checkout module" from the "Cvs Admin" menu.  Specify "locale" for the module name.

CVS command line

You can also check out the repository anonymously on UNIX using the following commands, after first setting your CVSROOT to point to the CLDR repository:

  1. export CVSROOT=:pserver:anoncvs@oss.software.ibm.com:/usr/cvs/icu 
  2. cvs login CVS password: anoncvs 
  3. cvs checkout -P locale

This will extract the absolute latest version of all files from the repository. If you want to extract a particular release, say 1.2.3, pass a -r TAG option to cvs checkout. For example:

cvs checkout -P -r release-1-2-3 locale

You should then have a working repository.

  • The CVS tag for the 1.1 release is "release-1-1".
  • The CVS tag for the 1.0 release is "release-1-0".

Repository Organization

In the descriptions below, <locale_data > is the full path name of the locale data root directory (the top level directory from the distribution archives) in your file system. The directories are named to reflect the source platform from which they are collected; for e.g. locale data collected from ICU is in the directory named “icu”.

The following directories contain source code and data files:

<path>/locale/ The repository containing locale data from different platforms.
<path>/locale/<platform_name> Contains locale data collected from the specific platform denoted by the <platform_name>
<path>/locale/<platform_name>/main The locale data in XML format, e.g.: locale/icu/main/en.xml, locale/icu/main/fr_FR.xml.
<path>/locale/<platform_name>/collation The collation data in XML format, e.g.: locale/icu/collaiton/en.xml, locale/icu/collation/fr_FR.xml.
<path>/locale/diff/main Contains the comparison charts for main data in the locales. The charts show the differences in tabular format
<path>/locale/diff/collation Contains the comparison charts for collation data in the locales.

The XML source data is organized in the following directories. The xml files can be divided into main/ and collation/.

aix/
common/
hp/
ibmjdk/
icu/
linux/
open_office/
solaris/
sunjdk/
windows/

Note: the data for all but common is provided for comparison only, and is not to be viewed as authoritative or referenceable.
