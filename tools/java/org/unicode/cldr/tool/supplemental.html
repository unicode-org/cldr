This set of tables shows the supplemental data contained in CLDR. This is data that is not language-based, but is required for use with the language-based data in CLDR. If you are vetting this data, please look over the tables to see how they are structured. Look for the countries and languages that you are familiar with, and see if the data is correct. If not, or if there is missing data, please file a bug. When you do so, please include references supporting the request.

Instructions

Currencies

  • The Territory → Currency table shows which countries used which currencies, during which periods.
  • The Currency Format Info table shows information organized by currency. The digits column shows the number of digits to use; if there is special rounding (such as for CH), that is in parentheses. The Countries column shows which countries the currency is — or has been — used in, officially.

Languages, Scripts, and Territories

The Language and Territories tables show modern official and major commercial languages for the territories in question. Cases where the language is used, but is neither official nor a major commercial language are shown with *.

The Language and Scripts tables do the same for scripts.

In each case, the Not Represented subtables list territories, scripts, and languages that are not represented in the above tables, for cross-checking. There are two types of cases:

  • Languages or scripts that are not in modern use, such as "[Glag] Glagolitic"
  • Other cases are listed there because we don't yet have the data

Note: this data is incomplete: the focus has been on collecting data for languages represented in CLDR.

Other Territory Data

The Other Territory Data table shows other information about territories, such as which calendars are in common use (other than the Gregorian calendar) in which territories.

Territories and Timezones

The Territory Containment table shows the organization of territories and regions according to UN M.49, starting with the World. (CLDR supplements this table with the QO code for outlying areas that would not otherwise be included.) As the last column, the timezone IDs for that country are listed.

The Windows-Tzid table shows the mapping from Windows timezone IDs to the standard TZIDs.

Character Fallback

The Character Fallback Substitutions table shows recommended fallbacks for use when a charset or supported repertoire does not contain a desired character, using the data from characters.xml. There is more than one possible fallback: the recommended usage is that when a character value is not in the desired repertoire, the explicit substitutes from characters.xml are tested one by one against the repertoire, with the first substitute wholly in the repertoire being substituted for the value in the output. If no explicit substitute is found, then toNFC(value) is tried; if that fails then toNFKC(value) is tried. (The Explicit, NFC, and NFKC substitutes are shown in the chart by different colors).

Note that the character fallbacks do lose information, and should not be used where there is a viable alternative, such as HTML escapes.

Metadata

Aliases show how to map deprecated codes or aliases onto the ones that should be used to access CLDR data. Most other metadata is not shown in tables; the source data should be consulted.

Note: Codes are shown in brackets before or after the English name, eg "Vanuatu [VU]"
